Optional delayMs

delayMs?: number
  • Defined in src/lib/utils/retry-on-conflict.ts:32

The number of milliseconds to wait before retrying a failed request. This will be increased exponentially {@see CommercetoolsApi.calculateDelay}. Defaults to 100.

Optional jitter

jitter?: boolean
  • Defined in src/lib/utils/retry-on-conflict.ts:47

If enabled, adds a random element to the exponential increase in retry time. See the following url for more details: https://aws.amazon.com/blogs/architecture/exponential-backoff-and-jitter/ We utilise the 'full' jitter + plus an additional decaying variance.

Optional maxRetries

maxRetries?: number
  • Defined in src/lib/utils/retry-on-conflict.ts:39

The maximum number of times that we'll call the executeFn function before returning the error caught from the last failure. Defaults to 3.
